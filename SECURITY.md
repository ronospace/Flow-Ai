# Security Policy

## Supported Versions

Security updates are provided for the latest production release and the active release branch.

| Version | Supported |
| -------- | --------- |
| Latest production release | ✅ |
| Active release branch | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues or discussions.

Use one of the following methods:

- GitHub Private Vulnerability Reporting (preferred)
- Security contact: flowai.zyraflow@gmail.com

Please include:

- A clear description of the issue
- Steps to reproduce
- Affected versions
- Impact assessment
- Proof of concept, if available

We will acknowledge reports, investigate them responsibly, and coordinate disclosure after a fix has been prepared.

## Scope

Examples include:

- Authentication or authorization flaws
- Personal or health data exposure
- Encryption or secure storage issues
- Payment or subscription validation weaknesses
- Dependency or supply-chain vulnerabilities
- Privilege escalation
- Privacy control bypasses

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing a vulnerability. Do not access, modify, or retain user data beyond what is necessary to demonstrate the issue.
